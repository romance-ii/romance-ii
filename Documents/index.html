“Romance” Game System

This is the development home of the “Romance” game system, based around the Appius Claudius Cæcus game server and Publius Vergilius Maro game client framework.

What is here?

Here, you can:

  • Download the Romance source code
  • With permission, access the Subversion repository to contribute to Romance
  • Read the current JavaDoc documentation, and Designer and Developer documents, as well.

What does it do?

Appius Claudius Cæcus is a Java-based videogame server. It handles communications between players, enforcing game-specific rules, and managing things like inventory and currency.

Appius supports access using a JSON-based protocol called Infinity. It also supports a subset of the communications protocol of Smart Fox Server Pro for compatibility, but the feature sets do not align (SFS has features which Appius does not implement, and vice-versa) so there are still some client changes necessary to integrate with Appius. Communications using Infinity can be conducted in TCP stream-oriented fashion, in a TCP batched-command-set mode, or using a WebSocket.

Publius Vergilius Maro is a Java-based client-side architecture for communicating with Appius and playing a game (e.g. drawing a scene). It is being developed currently for use with Android applications, and can also be compiled into native binary applications for many platforms using GCC (GCJ), into JavaScript (EcmaScript) using Google Web Toolkit (GWT), and is intended to be potentially portable to Java Mobile Edition and other Java-based application hosts.
